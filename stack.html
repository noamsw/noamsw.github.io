Stack – The epitome of a LIFO data structure

The stack is the most classic example of a LIFO – Last In, First Out data structure.
The stack is used for many different applications in computer science. A stack is a one-ended linear DS, that models real world stacks, wherein elements are added and removed to the top of the pile.
This is why it is considered LIFO, the last element added is the first element removed.

An ADT of stacks commonly includes these five methods:

  • Push() - a method which adds a single element to the top of the stack, O(1)

  • Pop() - a method which removes the topmost element from the stack, O(1)

  • Peek() - a method which returns the topmost element of the stack, without removal, O(1)

  • Size() – a method which returns the number of elements currently in the stack, O(1)

  • depending on the implementation chosen, another common method is:

  • IsFull() - a method which returns true if the stack is filled to capacity, O(1)

  • My implementation uses a linked list and therefore does not require this method, as we can grow the stack as we wish.
