Quick Sort

A linear sorting algorithm

Radix sort sorts elements digit by digit, and can be used to sort positive integers or fixed length strings.
Radix sort sorts the elements iteratively from least significant to most significant.
When the order is tied, the prior order is maintained.
Once you complete the ordering of all digits, you have sorted the elements.

Time Complexity Analysis

Let us denote the following variables::

n - the number of elements to be sorted.
k - the number of digits in the maximum number (the number of passes).
b - the base (usually 10 for decimal numbers)
In order to sort the digits we use counting sort.
Counting sort is known to have a time complexity of n + b for n elements in base b.
In radix sort we go over the numbers one digit at a time, therefore we will have to go over the numbers K times.
Therefore the time complexity is O(k * (n + b)).
